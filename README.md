#1 맥도날드 춘식이 콜라보 아나모픽 3D 광고 영상
-
- 개발 기간: 2025.08.29 ~ 2025.09.04 (유니티 부트캠프 13기 팀 프로젝트)
- 프로젝트 역할: 팀장, 3D 아티스트 & 연출
- 담당 업무
  - 애니메이션 콘티 제작: 광고 영상의 전체적인 스토리보드와 장면 전환, 카메라 워크 등 시각적 흐름을 기획.
  - 3D 모델링 (Blender): 아나모픽 착시 효과의 기준이 되는 핵심 구조물(프레임)과 브랜드 로고(맥도날드)를 Blender를 사용하여 직접 모델링.
  - 프로젝트 총괄: 팀의 개발 방향성 설정, 업무 분배, 일정 조율 등 프로젝트 전반을 리드.
- 사용 기술 및 환경
  - Rendering & Animation: Unity, C#
  - 3D Modeling: Blender
  - AI-Assisted Modeling: Meshy
  - Projection Mapping: MadMapper
- 상세 내용
  - 맥도날드와 '춘식이' 캐릭터 콜라보를 모티브로, 옥외 광고판에서 흔히 볼 수 있는 아나모픽(Anamorphic) 3D 광고 영상을 제작하는 5인 팀 프로젝트에서 팀장 및 메인 연출 역할을 수행했습니다.
  - 저의 핵심적인 역할은 프로젝트의 시작부터 끝까지 시각적인 결과물을 책임지는 것이었습니다. 먼저, 영상의 전체적인 흐름을 담은 애니메이션 콘티를 직접 제작하여 팀원들과의 비전을 공유했습니다. 이후 Blender를 사용하여 아나모픽 효과의 기반이 되는 프레임 구조물과 맥도날드 로고를 3D로 모델링했습니다.
  - 최종적으로 Unity에서 렌더링한 영상을 프로젝션 맵핑 툴인 MadMapper에 적용하여 실제 아나모픽 광고처럼 보이도록 구현했습니다.
- 프로젝트를 통해 얻은 경험
  - 이번 프로젝트를 통해 3D 모델링 툴(Blender)과 Unity 타임라인에 대한 깊은 이해를 얻을 수 있었습니다. 특히, 정해진 프레임 안에서 착시 효과를 극대화해야 하는 아나모픽 기법의 특성을 고려하며 애니메이션을 연출하는 과정은 기술과 예술적 감각을 동시에 요구하는 값진 경험이었습니다. 게임 개발뿐만 아니라 인터랙티브 미디어 아트, 광고 등 다양한 분야로 기술을 확장할 수 있는 가능성을 보았습니다.
- 시연 영상 링크

#2 VR 멀티플레이 숫자야구 게임
-
- 개발 기간: 2025.08.14 ~ 2025.08.28 (유니티 부트캠프 13기 팀 프로젝트)
- 프로젝트 역할: 팀장, 백엔드/네트워크 시스템 개발 담당
- 담당 업무
  - Unity Gaming Services (UGS) 총괄: 멀티플레이어 기능 구현에 필요한 UGS의 모든 서비스를 전담하여 설계하고 연동.
  - 인증 시스템 (Auth): 플레이어의 익명 로그인 및 기기 인증 처리를 구현하여 서버에 안전하게 접속할 수 있는 기반 마련.
  - 매치메이킹 시스템 (Lobby): 플레이어가 게임 방을 생성하거나, 조건에 맞는 방을 자동으로 탐색하고 참가하는 로비 및 매치메이킹 로직 전체를 구현.
  - 실시간 데이터 전송 (Relay): P2P 연결이 어려운 네트워크 환경(방화벽, NAT)에서도 안정적인 게임 플레이가 가능하도록 UGS Relay 서버를 통한 데이터 중계 시스템 구축.
  - 3D 공간 음성 채팅 (Vivox): VR 환경의 몰입감을 높이기 위해 플레이어의 머리 위치와 거리에 따라 소리가 다르게 들리는 3D 공간 음성 채팅 기능 연동.
- 사용 기술 및 환경
  - Game Engine: Unity (VR), Netcode for GameObjects
  - Backend Services (UGS): Auth, Lobby, Relay, Vivox
  - Version Control: Git, GitHub
- 상세 내용
  - VR 환경에서 즐기는 실시간 멀티플레이 숫자야구 게임 프로젝트에서, 저는 Unity Gaming Services(UGS)를 활용한 전체 백엔드 및 네트워크 시스템 구축을 전담했습니다. 별도의 서버 코드를 직접 개발하는 대신, UGS가 제공하는 강력한 서비스들을 통합하여 짧은 기간 내에 안정적이고 확장성 있는 멀티플레이어 환경을 만드는 것을 목표로 했습니다.
  - 프로젝트의 핵심은 플레이어가 원활하게 만나서 게임을 즐길 수 있는 온라인 인프라를 구축하는 것이었습니다. 이를 위해 저는 다음의 흐름을 직접 설계하고 구현했습니다.
1) Auth 서비스를 통해 플레이어가 게임에 접속하면 고유 ID를 부여받습니다.
2) 게임 시작을 눌렀을 때 생성된 Lobby 가 있으면 자동으로 참가하고, 없으면 새로운 Lobby 를 만듭니다.
3) 매칭이 완료되면, 로비의 호스트는 Relay 서버에 가상 게임 세션을 생성하고 접속 코드(Join Code)를 생성합니다.
4) 클라이언트는 이 코드를 받아 Relay 서버에 접속하여, 모든 플레이어가 하나의 게임 씬에서 만날 수 있게 됩니다.
5) 게임이 시작되면 Vivox Voice 서비스가 활성화되어, 플레이어들은 VR 공간 안에서 서로의 목소리를 들으며 실시간으로 소통할 수 있습니다.
- 문제 해결 과정 (UGS 통합 안정화)
  - 문제 정의: 프로젝트 초기, UGS의 여러 서비스를 동시에 연동하는 과정에서 인증 토큰이 만료되거나, 로비 데이터가 충돌하는 등 불안정한 문제가 간헐적으로 발생했습니다. 특히 방화벽 환경에 따라 Relay 서버 접속 실패율이 높아지는 현상이 있었습니다.
  - 해결 전략
  1) 데이터 교환 최적화 (Lobby): 로비에서 주고받는 데이터를 접속 코드(Join Code)와 준비 상태(Ready) 등 필수 정보로 최소화하고, 데이터 버전을 체크하는 로직을 추가하여 정보가 덮어씌워지는 문제를 해결했습니다.
  2) 접속 실패 대응 (Relay): 플레이어의 네트워크 환경(Ping)을 미리 측정하여 최적의 지역 서버에 접속하도록 하고, 접속 실패 시 지수 백오프(Exponential Backoff) 알고리즘에 따라 자동으로 재접속을 시도하도록 구현하여 세션 연결 성공률을 높였습니다.
- 결과 및 성과
  - 복잡한 서버 개발 없이도 UGS만으로 "로그인-매치메이킹-게임시작-음성채팅"으로 이어지는 매끄러운 멀티플레이 경험을 성공적으로 구현할 수 있었습니다.
  - 프로젝트 시연 영상을 개인 인스타그램에 공유하여 조회수 2,700회, 총 시청 시간 9시간 18분을 달성하며 잠재 유저들의 높은 관심을 확인했습니다. 이를 통해 직접 개발한 콘텐츠에 대한 유저들의 생생한 반응을 살펴보는 즐거움을 경험하고, 사용자 피드백의 중요성을 체감할 수 있었습니다.

#3 오버워치 '정켄슈타인의 복수' 모바일 팬게임
- 
- 개발 기간: 2025.06.23 ~ 2025.07.10 (유니티 부트캠프 13기 팀 프로젝트)
- 프로젝트 역할: 팀장
- 담당 업무
  - 오브젝트 풀링(Object Pooling) 시스템 설계 및 구현: 게임의 핵심 최적화 담당. 반복적으로 생성/삭제되는 적(좀닉)과 투사체(발사체)를 미리 생성하고 재활용하는 시스템을 구축하여 메모리 부하를 줄이고 게임 성능을 안정적으로 확보.
  - PlayFab 연동 랭킹 시스템 구현: 백엔드 서비스인 PlayFab을 활용하여 유저의 게임 결과를 서버에 기록하고, 순위를 표시하는 온라인 랭킹 보드 기능 개발.
  - 투사체 시스템 개발: 플레이어와 적 캐릭터가 사용하는 다양한 총알, 스킬 발사체의 물리 로직 및 생명주기 관리.
  - UI/UX 기획 및 구현: 모바일 환경에 최적화된 게임 내 인터페이스 설계 및 제작.
  - 팀 프로젝트 총괄 및 Git 버전 관리: 팀의 전체 개발 일정 조율, 업무 분배, 그리고 GitHub를 통한 소스 코드 버전 관리 총괄.
- 사용 기술 및 환경:
  - Game Engine: Unity, C#
  - Backend: PlayFab (랭킹 시스템)
  - Core System: Unity Input Systmem, Object Pooling
  - Version Control: Git, GitHub
- 상세 내용
  - 블리자드의 '오버워치' PVE 콘텐츠를 모바일 환경으로 이식하는 5인 팀 프로젝트에서 팀장 역할을 맡았습니다. 프로젝트 기획부터 최종 빌드까지 전체 개발 파이프라인을 이끌며, 특히 게임의 성능 최적화와 핵심 시스템 구현에 중점적으로 기여했습니다.
  - 저의 핵심 담당 업무는 오브젝트 풀링 시스템 구축이었습니다. 수많은 적과 투사체가 실시간으로 등장하는 디펜스 장르의 특성상, Instantiate와 Destroy 함수의 잦은 호출로 인한 성능 저하가 예상되었습니다. 이를 해결하기 위해, 게임 시작 시 필요한 오브젝트들을 미리 생성해두는 풀(Pool)을 만들고, 필요할 때마다 꺼내 쓰고 반납하는 재활용 시스템을 설계했습니다. 이 시스템 덕분에 게임의 프레임 드랍 현상을 크게 개선하고 안정적인 플레이 환경을 제공할 수 있었습니다.
  - 또한, PlayFab을 연동하여 서버 기반의 랭킹 시스템을 구현하여 싱글 플레이 게임에 경쟁 요소를 더했습니다. 이 외에도 다양한 투사체의 로직을 개발하고, 팀원들과 협력하여 UI/UX를 완성했습니다.
- 문제 해결 과정 (오브젝트 풀링)
  - 문제 정의: 초기 개발 단계에서 적 웨이브가 시작되면 수많은 총알과 적 캐릭터가 생성 및 삭제를 반복하면서 심각한 프레임 드랍과 끊김 현상(Garbage Collection)이 발생했습니다.
  - 해결 전략:
    1. 재사용 설계: 적(Zomnic)과 투사체(Projectile)를 위한 각각의 오브젝트 풀 매니저(Pool Manager)를 싱글톤 패턴으로 구현했습니다.
    2. 자동 반납 로직: 풀에서 꺼내 쓴 오브젝트가 수명이 다하거나 비활성화될 때(예: 적이 죽거나, 투사체가 벽에 부딪혔을 때), 자동으로 풀에 반납되도록 코루틴(Coroutine)과 인터페이스를 활용하여 시스템을 자동화했습니다.
    3. 확장성 확보: Dictionary를 사용하여 다양한 종류의 적과 투사체를 ID(정수) 값으로 관리할 수 있도록 설계하여, 추후 새로운 캐릭터나 무기가 추가되더라도 쉽게 확장할 수 있는 구조를 만들었습니다.
  - 결과: 오브젝트 풀링 시스템 도입 후, 게임의 평균 FPS(초당 프레임)가 안정적으로 유지되었으며, 메모리 사용량을 크게 절감하여 쾌적한 모바일 플레이 경험을 구현하는 데 성공했습니다.

#4 대학 졸업논문 프로젝트
-
- 졸업 논문명: D램 커패시터 성능 향상을 위한 최적화된 ZrO₂-HfO₂-Al₂O₃ 구조 탐색 및 전기적 특성 평가 연구
- 연구 기간: 2022.08.15 ~ 2022.12.17
- 담당 업무 및 역할:
  - DRAM 커패시터 및 High-k(고유전율) 신소재 관련 선행 연구 분석
  - ALD(원자층 증착법) 장비를 이용한 ZrO₂-HfO₂-Al₂O₃ 박막 증착 실험 설계 및 수행
  - Probe Station을 활용하여 제작된 소자의 C-V, I-V 등 전기적 특성 측정 및 분석
  - Ellipsometer, SEM 등을 통한 박막의 물리적 특성 분석
  - 실험 결과 데이터화 및 논문 작성
- 사용 기술 및 환경:
  - 증착 장비: ALD(Atomic Layer Deposition)
  - 측정/분석 장비: Ellipsometer, Probe Station, Semiconductor Analyzer (Keithley 4200-SCS), SEM
  - 핵심 소재: ZrO₂, HfO₂, Al₂O₃ (High-k Dielectrics)
- 상세 내용
  - DRAM의 고집적화에 따라 발생하는 커패시터의 정전용량(Capacitance) 감소 문제를 해결하기 위해 진행한 연구입니다. 기존의 구조적 개선 방법의 한계를 극복하고자, 높은 유전율을 가진 신소재 ZrO₂와 HfO₂를 활용하여 소재 측면에서 성능을 향상시키는 것을 목표로 삼았습니다.
  - ALD(원자층 증착법) 기술을 사용하여 ZrO₂와 HfO₂ 박막의 층수와 두께를 다르게 조합한 여러 유전체 구조를 설계하고 직접 증착했습니다. 또한, 누설 전류 특성을 개선하기 위해 박막 중간과 상부에 Al₂O₃를 삽입하여 안정성을 높였습니다.
  - 실험 결과, 박막을 더 얇고 다층 구조로 만들수록 Grain size가 감소하여 누설 전류 특성이 향상되는 장점을 확인했습니다. 비록 유전율은 소폭 감소했지만, DRAM 동작에 필수적인 반강유전성(Anti-ferroelectric) 특성을 유지하며 낮은 등가산화막두께(EOT) 값을 달성하여 차세대 DRAM 커패시터 소재로서의 적용 가능성을 증명했습니다.
- 문제 해결 과정
  - 문제 정의: DRAM 소자가 미세화될수록 커패시터의 물리적 면적이 줄어들어 정전용량이 감소하고, 이는 데이터 저장 능력을 저하시킵니다. 또한, 유전막 두께를 줄이면 터널링 현상으로 누설 전류가 증가하는 문제가 발생합니다.
  - 해결 전략:
    1. SiO₂ 대비 월등히 높은 유전율을 갖는 ZrO₂와 HfO₂를 유전막 소재로 채택하여 정전용량 확보.
    2. 두 소재를 나노미터 단위의 다층 구조로 설계하여 높은 유전율을 갖는 결정상(Tetragonal)을 안정적으로 형성하고, 동시에 Grain 크기를 제어하여 누설 전류 경로를 최소화.
    3. 절연 특성이 우수한 Al₂O₃ 박막을 중간층으로 삽입하여 누설 전류를 효과적으로 억제.
  - 결과: 위 전략을 통해 낮은 누설 전류(<10⁻⁷A/cm²) 기준을 만족하면서도 0.63nm 수준의 낮은 EOT(등가산화막두께) 값을 구현하여, DRAM의 집적도를 한 단계 더 높일 수 있는 새로운 유전체 구조를 성공적으로 제안했습니다.
